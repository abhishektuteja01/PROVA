1. MODEL PURPOSE AND INTENDED USE (CS-01)

The GARCH(1,1) Volatility Forecasting Model ("the Model") produces one-day-ahead and multi-step-ahead forecasts of annualized return volatility for equity indices and single-name equities. The Model supports:

- Value-at-Risk (VaR) calculations for the equity trading desk's market risk framework
- Volatility input for the equity options pricing system (when implied volatility is unavailable)
- Risk-weighted asset calculations under the standardized approach for market risk

The Model is approved for: equity indices (S&P 500, NASDAQ-100, Russell 2000, Euro Stoxx 50) and single-name equities with at least 2 years of daily return history.

The Model must NOT be used for: fixed income volatility, FX volatility, commodity volatility, options on volatility (e.g., VIX derivatives), or any asset with non-continuous price series.

2. THEORETICAL AND MATHEMATICAL FRAMEWORK (CS-02)

The Model implements the Generalized Autoregressive Conditional Heteroskedasticity GARCH(1,1) specification introduced by Bollerslev (1986).

The return process is:

    r_t = mu + epsilon_t
    epsilon_t = sigma_t * z_t,  z_t ~ N(0,1)

The conditional variance equation is:

    sigma_t^2 = omega + alpha * epsilon_{t-1}^2 + beta * sigma_{t-1}^2

where:
- omega > 0 is the long-run variance intercept
- alpha >= 0 is the ARCH coefficient (reaction to shocks)
- beta >= 0 is the GARCH coefficient (persistence)
- alpha + beta < 1 is required for covariance stationarity

The long-run (unconditional) variance is:

    sigma_LR^2 = omega / (1 - alpha - beta)

Parameters are estimated via maximum likelihood estimation (MLE) assuming normally distributed innovations. The log-likelihood function is:

    L(theta) = -(T/2) * ln(2*pi) - (1/2) * sum_{t=1}^{T} [ln(sigma_t^2) + epsilon_t^2 / sigma_t^2]

Optimization uses the BFGS algorithm with numerical gradients. Starting values are set using variance targeting: omega = (1 - alpha_0 - beta_0) * sample_variance, with alpha_0 = 0.05, beta_0 = 0.90.

Multi-step forecasts are generated using the recursive formula:

    E[sigma_{t+h}^2 | I_t] = sigma_LR^2 + (alpha + beta)^h * (sigma_t^2 - sigma_LR^2)

3. KEY ASSUMPTIONS (CS-03)

Assumption 1 — Normally distributed innovations: The standardized residuals z_t are assumed to follow a standard normal distribution. This assumption enables closed-form MLE and simplifies VaR calculations.

Assumption 2 — Covariance stationarity: The constraint alpha + beta < 1 ensures the variance process is mean-reverting. The Model's forecasts converge to the unconditional variance at long horizons.

Assumption 3 — Symmetric volatility response: The standard GARCH(1,1) treats positive and negative shocks of equal magnitude identically. The Model does not capture the leverage effect (negative returns increase volatility more than positive returns of the same magnitude).

Assumption 4 — Single-asset framework: The Model estimates volatility for each asset independently. It does not model cross-asset correlation dynamics or volatility spillovers.

Assumption 5 — Constant parameters: The GARCH parameters (omega, alpha, beta) are assumed to be constant over the estimation window. In practice, parameters may shift during regime changes.

4. ASSUMPTION LIMITATIONS AND BOUNDARIES (CS-04)

Normal distribution limitation: Equity returns exhibit fat tails (excess kurtosis of 4-8 for daily returns vs. 3 for normal). The Model underestimates the probability of extreme moves, causing VaR to be breached more frequently than the confidence level implies. Backtesting shows 99% VaR is exceeded on approximately 2.1% of days (vs. expected 1.0%).

Symmetric response limitation: The absence of a leverage effect means the Model underestimates volatility following market declines and overestimates volatility following market rallies. Empirically, a 2% market decline increases next-day volatility by approximately 15%, while a 2% rally decreases it by only 8%. The GARCH(1,1) predicts a symmetric 11% change.

Parameter instability: Rolling 2-year estimation windows show that the persistence parameter (alpha + beta) ranges from 0.93 to 0.99 over the 2015-2024 period. During the COVID-19 crash (March 2020), the estimated alpha temporarily spiked to 0.25 (from the typical 0.05-0.10 range), indicating the model's parameters are not truly constant.

Regime dependence: The Model assumes a single volatility regime. In reality, volatility alternates between low-volatility (VIX < 15) and high-volatility (VIX > 25) regimes with different dynamics. During regime transitions, the Model's forecasts lag the actual volatility shift by 5-10 trading days.

- Multivariate volatility or correlation forecasting

7. KNOWN MODEL WEAKNESSES (CS-07)

Weakness 1 — Fat tail underestimation: The normal distribution assumption causes systematic VaR underestimation. A Student-t or GED distribution would better capture tail risk but introduces additional estimation complexity.

Weakness 2 — No leverage effect: The symmetric GARCH specification ignores the empirically documented negative correlation between returns and volatility. GJR-GARCH or EGARCH specifications would address this but are not currently implemented.

Weakness 3 — Slow adaptation to regime changes: The exponential decay of shock impact (governed by alpha + beta ≈ 0.95) means the Model takes approximately 20 trading days to fully incorporate a new volatility regime. This is a structural feature of the GARCH(1,1) specification.

Weakness 4 — Multi-step forecast degradation: The recursive multi-step formula converges to the unconditional variance, losing signal rapidly. By day 10, the forecast is approximately 60% converged to the long-run level. For 10-day VaR, this means the Model's forecasts are more reflective of long-run average volatility than current conditions.

1. BACKTESTING METHODOLOGY (OA-01)

Backtesting is performed on a rolling basis using the last 3 years of data (January 2022 through December 2024, 756 trading days).

For each day, the Model's one-day-ahead volatility forecast is compared against the realized daily return. Two backtesting frameworks are used:

Framework 1 — VaR backtesting: The Model's 95% and 99% VaR forecasts (computed as sigma_forecast * z_alpha) are compared against actual losses. Exception days (losses exceeding VaR) are counted and tested against the expected rate.

Framework 2 — Volatility forecast accuracy: The Model's volatility forecast is compared against realized volatility estimated as the 20-day rolling standard deviation of returns, centered on the forecast date.

Backtesting universe: S&P 500 index (primary benchmark), plus 50 individual S&P 500 constituents selected to cover all 11 GICS sectors.

2. PERFORMANCE METRICS (OA-02)

95% VaR Exception Rate: 6.1% (expected: 5.0%). The Model slightly under-covers at 95%, consistent with the known fat-tail limitation. Over 756 days, observed 46 exceptions vs. expected 38.

99% VaR Exception Rate: 2.1% (expected: 1.0%). More significant under-coverage at 99%, again due to fat tails. Observed 16 exceptions vs. expected 8. This places the Model in the Basel yellow zone (5-9 exceptions per 250 days).

Volatility Forecast RMSE: 3.2 percentage points (annualized). Mean absolute forecast error: 2.4 pp.

Mincer-Zarnowitz Regression: Realized volatility regressed on forecast volatility yields slope = 0.84, intercept = 2.8%, R-squared = 0.61. The slope below 1.0 indicates the Model tends to underestimate high-volatility periods and overestimate low-volatility periods.

QLIKE Loss Function: 0.043 (lower is better). This loss function is robust to noise in realized volatility estimation. The GARCH(1,1) QLIKE of 0.043 outperforms the historical volatility benchmark (0.058).

3. BENCHMARKING (OA-03)

Benchmark 1 — Historical Volatility (20-day rolling): RMSE = 4.8pp (vs. Model's 3.2pp). QLIKE = 0.058. The GARCH model substantially outperforms naive historical vol, particularly during volatility transitions.

Benchmark 2 — EWMA (RiskMetrics, lambda=0.94): RMSE = 3.5pp. QLIKE = 0.048. The GARCH model modestly outperforms EWMA due to the mean-reversion property — EWMA lacks mean reversion, causing forecasts to remain elevated too long after volatility spikes.

Benchmark 3 — GJR-GARCH(1,1) (asymmetric model, not currently in production): RMSE = 2.9pp. QLIKE = 0.039. The asymmetric model outperforms the symmetric GARCH(1,1) by 9% on RMSE. This improvement is concentrated in post-crash periods where the leverage effect is most pronounced. The GJR specification is recommended for future model enhancement.

5. STRESS TESTING (OA-05)

Scenario 1 — COVID-19 Crash Replay (March 2020):
S&P 500 declined 34% in 23 trading days. VIX spiked from 14 to 82.
Model behavior: The GARCH forecast lagged actual volatility by 5-8 days during the initial crash. By day 10, the Model's forecast caught up to the elevated regime. During the peak stress (VIX > 60), the Model underestimated 1-day volatility by approximately 40%. VaR was breached on 8 of 23 crash days.

Scenario 2 — Flash Crash (May 6, 2010):
Instantaneous 9% decline and recovery within minutes.
Model behavior: The single extreme return causes a large one-day spike in the GARCH forecast, which then decays exponentially. The Model overpredicts volatility for the following 15 trading days as the shock decays through the system. This is a feature of the GARCH structure, not a bug, but it illustrates the Model's inability to distinguish transient events from sustained regime changes.

Scenario 3 — Prolonged Low Volatility (2017):
VIX averaged 11.1 for the year, with 52-week realized vol of 6.7%.
Model behavior: GARCH forecasts gradually converged to the low-vol regime over approximately 40 trading days. During this period, VaR estimates were conservatively high. The Model correctly adapted but with significant lag.

6. OUT-OF-SAMPLE TESTING (OA-06)

Model parameters were estimated on 2018-2022 data. Out-of-sample testing on 2023-2024:

- RMSE: 3.4pp (vs. 3.2pp in-sample) — 6% degradation, acceptable.
- 95% VaR exception rate: 5.8% (vs. 6.1% in-sample) — slight improvement out-of-sample.
- 99% VaR exception rate: 1.8% (vs. 2.1% in-sample) — still above expected 1.0%.
- Mincer-Zarnowitz R-squared: 0.58 (vs. 0.61 in-sample).

The out-of-sample period included the October 2023 bond market volatility spillover and the July 2024 yen carry trade unwind. The Model handled both events with forecast errors within 1.5x the normal range, recovering within 5 trading days.

Cross-asset hold-out: The Model was estimated on S&P 500 and tested on NASDAQ-100 (not in the estimation sample):
- RMSE: 4.1pp (vs. 3.2pp for S&P 500)
- The higher error reflects NASDAQ's greater concentration risk and higher beta to tech sector events

7. STATISTICAL VALIDATION (OA-07)

Kupiec POF Test (95% VaR): LR = 1.82, p-value = 0.18. PASS — exception rate consistent with 95% confidence level.

Kupiec POF Test (99% VaR): LR = 6.43, p-value = 0.011. MARGINAL FAIL at 1% significance, PASS at 5%. The elevated exception rate at 99% reflects the fat-tail limitation.

Christoffersen Independence Test: LR = 0.94, p-value = 0.33. PASS — VaR exceptions show no significant clustering.

Ljung-Box Test on Standardized Residuals: Q(20) = 18.4, p-value = 0.56. PASS — no significant autocorrelation in standardized residuals, confirming the GARCH(1,1) adequately captures volatility dynamics.

Ljung-Box Test on Squared Standardized Residuals: Q(20) = 24.1, p-value = 0.24. PASS — no remaining ARCH effects in squared residuals, confirming the model order is sufficient.

Jarque-Bera Test on Standardized Residuals: JB = 142.3, p-value < 0.001. FAIL — standardized residuals exhibit excess kurtosis (4.7 vs. expected 3.0), confirming the normal distribution assumption is violated. This is the known fat-tail limitation; a Student-t distribution would address it.
